# Module 1 — Introduction to Cybersecurity

## Defining Security

Security is the practice of ensuring the **confidentiality, integrity, and availability (CIA)** of information by protecting:

* Networks
* Devices
* People
* Data

This protection prevents **unauthorized access** and **criminal exploitation**.

---

# Security Analyst Role

Security analysts help protect organizations by monitoring systems, identifying threats, and responding to security incidents.

## Core Job Responsibilities

Security professionals are responsible for:

* **Protecting against internal and external threats**
* **Ensuring regulatory compliance** with laws and security standards
* **Maintaining business productivity** by preventing disruptions
* **Reducing financial risk** such as breach recovery costs, downtime, and regulatory fines
* **Maintaining brand trust** by preventing data breaches

---

## Security Analyst Responsibilities

Security analysts monitor and defend organizational systems and data.

Common responsibilities include:

1. **Protecting computer and network systems**
2. **Installing and managing prevention software**
3. **Conducting security audits**

### Security Audit

A **security audit** is a review of an organization's:

* Security records
* System activity
* Security policies
* Related documentation

Audits help identify **vulnerabilities and compliance gaps**.

# Social Engineering & Phishing

## Phishing

Phishing is a **social engineering attack** that uses psychological manipulation to trick people into revealing sensitive information.

Common tactics include creating a sense of:

* Urgency
* Fear
* Authority

Example:
An attacker sends an email pretending to be a company executive demanding immediate action.

---

## Phishing Indicators

Phishing messages often contain warning signs called **indicators**.

### Technical Indicators (Most Reliable)

These are the strongest signs of phishing.

Examples:

* Suspicious sender email domains
* Malicious or incorrect links

### Content Indicators

These are less definitive but still suspicious.

Examples:

* Poor grammar
* Spelling mistakes
* Generic greetings
* Unusual or urgent requests

# Security Operations Concepts

## Security Operations

Security operations focus on **monitoring, detecting, and responding to threats**.

Security analysts typically:

* Investigate alerts
* Respond to incidents
* Analyze suspicious activity

---

## Security Projects

Security engineers often work on **longer-term improvements**, such as:

* Creating new threat detections
* Improving existing detection systems
* Building security tools or infrastructure

---

## Playbooks

A **playbook** is a documented set of procedures used when responding to a security detection or incident.

Playbooks guide analysts by defining:

* Investigation steps
* Required data to review
* Response procedures

---

# Compliance & Security Frameworks

## Compliance

Compliance is the process of **following internal policies and external regulations**.

Organizations must comply with security standards to:

* Avoid regulatory fines
* Prevent data breaches
* Protect sensitive data

---

## Security Frameworks

Security frameworks provide **structured guidelines** for managing security risks.

They help organizations:

* Identify threats
* Reduce vulnerabilities
* Protect sensitive information

---

## Security Controls

Security controls are **safeguards implemented to reduce security risks**.

Examples include:

* Access controls
* Firewalls
* Monitoring systems

Security controls work alongside frameworks to strengthen security defenses.

---

## Security Posture

Security posture refers to an organization's overall ability to:

* Protect critical assets
* Defend against threats
* Respond to security incidents

A **strong security posture reduces risk and improves resilience**.

---

# Threats & Threat Actors

## Threat Actor

A **threat actor** is any individual or group that poses a security risk to an organization.

Examples include:

* Cybercriminals
* Hackers
* Malicious insiders
* Nation-state attackers

---

## Internal Threats

Internal threats come from trusted individuals such as:

* Current employees
* Former employees
* Third-party vendors
* Business partners

Internal threats may be:

**Intentional**

* Unauthorized data access
* Data theft

**Unintentional**

* Clicking malicious links
* Misconfiguring systems

---

# Types of Cybersecurity

## Network Security

Network security focuses on protecting an organization's network infrastructure.

This includes protecting:

* Data
* Systems
* Services
* Devices

The goal is to prevent **unauthorized access to network resources**.

---

## Cloud Security

Cloud security protects **data, applications, and infrastructure stored in cloud environments**.

### What is the Cloud?

The cloud is a network of **remote servers stored in data centers** that can be accessed through the internet.

Cloud services allow organizations to:

* Store data
* Run applications
* Access resources remotely

Cloud security ensures these resources are **properly configured and accessible only to authorized users**.

---

# Programming in Cybersecurity

Programming allows analysts to automate tasks and analyze data efficiently.

Examples of cybersecurity automation tasks:

* Searching lists of malicious domains
* Reviewing web traffic
* Detecting suspicious behavior
* Generating alerts

Automation helps security teams **work faster and respond to threats more effectively**.

---

# Intrusion Detection Systems (IDS)

An **Intrusion Detection System (IDS)** monitors system and network activity to detect potential intrusions.

IDS tools:

* Analyze traffic and system behavior
* Identify suspicious patterns
* Alert analysts when potential threats occur

---

# Threat Landscape Awareness

The **threat landscape** refers to the constantly evolving environment of cybersecurity threats.

Security analysts must stay informed about:

* Emerging malware
* New attack techniques
* Threat actor behaviors
* Industry vulnerabilities

This awareness helps analysts **detect threats more effectively**.

---

# Incident Response

Incident response is the process of **handling and recovering from cybersecurity incidents**.

Security analysts follow established procedures to:

* Investigate alerts
* Contain threats
* Remediate affected systems
* Prevent future incidents

---

# Sensitive Data

## Personally Identifiable Information (PII)

PII is any information that can be used to identify an individual.

Examples include:

* Full name
* Date of birth
* Physical address
* Phone number
* Email address
* IP address

---

## Sensitive Personally Identifiable Information (SPII)

SPII is a more sensitive category of PII that requires **stricter protection**.

Examples include:

* Social Security numbers
* Medical records
* Financial information
* Biometric data (facial recognition, fingerprints)

If stolen, SPII can cause **severe harm to individuals**.

---

# Identity Theft

Identity theft occurs when someone steals personal information to impersonate another person.

Threat actors often target **PII and SPII** during data breaches.

The most common goal is **financial gain** through fraud.
