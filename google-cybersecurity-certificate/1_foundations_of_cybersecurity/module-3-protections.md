# Module 3 — Protecting Against Threats, Risks, and Vulnerabilities

Cybersecurity professionals use frameworks, controls, and ethical practices to protect systems and sensitive data from security threats. These tools help organizations manage risk, maintain compliance, and build strong security defenses.

---

# Security Frameworks

A **security framework** is a structured set of guidelines used to build and maintain security programs within an organization.

Frameworks help organizations create **security lifecycles**, which are continuously evolving policies and procedures designed to reduce risk.

## Purpose of Security Frameworks

Organizations implement frameworks to:

* Protect **Personally Identifiable Information (PII)**
* Secure financial and sensitive data
* Identify security weaknesses
* Manage organizational risk
* Align security practices with business objectives

---

# Core Components of a Security Framework

Security frameworks typically include four major components.

## 1. Security Goals

Organizations define and document their security objectives.

Example:

An organization may establish a goal of complying with the **General Data Protection Regulation (GDPR)** to protect user data.

---

## 2. Guidelines

Guidelines provide instructions and policies that help organizations achieve their security goals.

Example:

To meet GDPR requirements, an organization might create policies that explain how user data requests must be handled.

---

## 3. Security Processes

Processes describe the operational steps required to implement security policies.

Example:

Security analysts may design procedures to ensure user requests—such as updating or deleting personal data—are verified and processed correctly.

---

## 4. Communication

Organizations must continuously monitor security activities and communicate findings.

Example:

A security analyst might detect a potential compliance issue and report it to management or a regulatory officer.

---

# Security Controls

**Security controls** are safeguards designed to reduce or eliminate specific security risks.

These controls may include:

* Security tools
* Policies
* Procedures
* Training programs

Example:

Using software to track whether employees complete required privacy training.

---

# Common Security Frameworks and Standards

Organizations rely on a variety of frameworks and regulations to manage cybersecurity risks and protect user data.

---

## CIA Triad

The **Confidentiality, Integrity, and Availability (CIA) Triad** is a foundational cybersecurity model used to guide security decisions and risk management.

### Confidentiality

Sensitive information should only be accessible to **authorized individuals**.

Example:

Access controls that restrict who can view certain data.

---

### Integrity

Data must remain **accurate, authentic, and trustworthy**.

Encryption and validation processes help maintain data integrity.

---

### Availability

Authorized users must be able to **access systems and data when needed**.

Example:

Maintaining reliable servers and backup systems.

---

## NIST Cybersecurity Framework (CSF)

The **NIST Cybersecurity Framework** is a voluntary framework that provides standards, guidelines, and best practices for managing cybersecurity risks.

Organizations use it to guide both **short-term and long-term security strategies**.

---

## FERC–NERC Regulations

The **Federal Energy Regulatory Commission (FERC)** and the **North American Electric Reliability Corporation (NERC)** regulate organizations involved in the North American power grid.

Organizations must:

* Prepare for potential cyber incidents
* Report security events
* Follow Critical Infrastructure Protection (CIP) standards

---

## FedRAMP

The **Federal Risk and Authorization Management Program (FedRAMP)** standardizes how cloud services are assessed, authorized, and monitored for security within the U.S. federal government.

Its purpose is to ensure consistent security practices across government agencies and cloud service providers.

---

## Center for Internet Security (CIS)

The **Center for Internet Security (CIS)** is a nonprofit organization that provides recommended security controls.

These controls help organizations:

* Strengthen system defenses
* Prevent common cyberattacks
* Respond effectively to security incidents

---

## General Data Protection Regulation (GDPR)

The **GDPR** is a European Union regulation that protects the privacy and personal data of EU residents.

Organizations must:

* Be transparent about how they collect and use personal data
* Notify affected individuals if a data breach occurs

Organizations typically have **72 hours** to report a breach involving EU citizens.

Failure to comply can result in significant fines.

---

## PCI DSS

The **Payment Card Industry Data Security Standard (PCI DSS)** is an international standard for protecting credit card information.

Organizations that store, process, or transmit credit card data must follow strict security requirements to reduce fraud.

---

## HIPAA

The **Health Insurance Portability and Accountability Act (HIPAA)** is a U.S. law that protects sensitive patient health information.

HIPAA is built around three primary rules:

1. Privacy Rule
2. Security Rule
3. Breach Notification Rule

Organizations must inform individuals if their **Protected Health Information (PHI)** is compromised in a breach.

Security professionals often work with the **Health Information Trust Alliance (HITRUST)** framework to help maintain HIPAA compliance.

---

## ISO Standards

The **International Organization for Standardization (ISO)** develops global standards related to technology, management, and security practices.

These standards help organizations improve areas such as:

* Operational processes
* Risk management
* Organizational efficiency

---

## SOC Reports

**System and Organization Controls (SOC)** reports evaluate how organizations manage data security and access controls.

SOC reports assess security practices across different levels of an organization, such as:

* Associates
* Supervisors
* Managers
* Executives
* Vendors

These reports help evaluate financial integrity, privacy, and data protection practices.

---

# Security Ethics

Security professionals must follow ethical principles when protecting systems and investigating incidents.

Ethical behavior helps maintain trust, ensure fairness, and protect individuals' rights.

---

## Ethical Security Principles

### Confidentiality

Security professionals must protect sensitive information and ensure it is only accessible to authorized users.

---

### Privacy Protection

Organizations must safeguard personal information and prevent unauthorized use or exposure of private data.

---

### Legal Responsibility

Security professionals must follow laws and regulations that govern cybersecurity practices.

Good ethical behavior includes:

* Remaining unbiased
* Relying on evidence
* Maintaining transparency
* Continuing professional development

---

# Ethical Concepts in Cybersecurity

## Counterattacks

A **counterattack** occurs when an organization attempts to retaliate against a threat actor.

Most organizations avoid counterattacks because they can:

* Escalate conflicts
* Cause unintended damage
* Create legal risks

### United States Laws

In the U.S., launching cyber counterattacks is generally illegal for private organizations.

Organizations are allowed to **defend themselves**, but retaliation is typically restricted to:

* Federal government agencies
* Military personnel

Counterattacks may also cause serious international consequences if state-sponsored attackers are involved.

---

### International Perspective

International guidance from the **International Court of Justice (ICJ)** suggests counterattacks may only occur under strict conditions:

* The response targets only the original attacker
* The goal is to stop the attack
* The response does not escalate the conflict
* The effects of the response can be reversed

# Key Security Terms

## Asset

An **asset** is any resource that has value to an organization.

Examples include:

* Data
* Systems
* Networks
* Intellectual property

---

## Availability

Availability means that systems and data remain accessible to **authorized users when needed**.

---

## Compliance

Compliance refers to following internal policies and external regulations related to security and data protection.

---

## Protected Health Information (PHI)

**PHI** refers to information related to an individual's physical or mental health.

Examples include:

* Medical records
* Health insurance information
* Treatment history

---

## Security Architecture

Security architecture is the design of systems, tools, and processes used to protect an organization from threats.

It defines how different security components work together.

---

## Security Governance

Security governance refers to the policies, leadership decisions, and management practices that guide an organization's cybersecurity strategy.

Governance helps ensure security efforts align with organizational goals and regulatory requirements.
